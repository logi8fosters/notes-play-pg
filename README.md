# notes-play-pg

这是一个用于归档和发布独立 HTML 页面的仓库。仓库中的每个页面均为独立的静态文件，不依赖于特定域名或网站环境，适合作为个人笔记、实验页面或简单展示的存档。

## 项目简介

本仓库的主要目的是将多个零散的 HTML 页面集中存放，便于版本管理、回顾与分享。所有页面均为纯前端实现，无需后端服务即可直接运行。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（如样式、脚本、图片）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个 HTML 文件均为一个独立页面，可直接通过浏览器打开。
- `assets/`：存放页面共用的样式、脚本、图片等资源，避免重复。

## 页面归档说明

- 每个页面以独立文件形式存放，命名尽量反映内容或用途。
- 页面之间不存在耦合关系，可单独添加、修改或删除。
- 如果页面需要引用外部资源（如 CDN），请确保资源链接稳定，或将其下载至 `assets/` 目录。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保引用路径正确。
- 修改页面：直接编辑对应 HTML 文件，建议在提交时附带简要说明。
- 删除页面：移除文件后，请检查是否有相关链接或引用需要同步清理。
- 不建议在本仓库中存放包含敏感信息或需要后端交互的页面。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 下的任意 HTML 文件。
3. 也可通过 GitHub Pages 或其他静态托管服务发布（需自行配置）。

## 许可

本仓库内容仅供参考与学习。如有需要，请参考各页面内附带的许可声明（如有）。
